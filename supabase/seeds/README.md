# Deterministic reference fixtures

Files in this directory contain reproducible, non-sensitive reference data for
blank local rebuilds. They never contain player, submission, identity, audit,
or operational rows.

`smite2-gods.sql` and `smite2-items.sql` are generated by
`diese-tech/smite-content-sync`. That project opens a review PR here instead
of writing to `sal-site` or production. Merging a fixture update alone does
not apply it to production; promote an approved content change through a
forward database migration and protected release.

The item feed owns stable IDs, names, source URLs, active state, and source
timestamps. It deliberately leaves `image_url` and `metadata` unchanged on
existing rows until those fields have canonical upstream sources.
